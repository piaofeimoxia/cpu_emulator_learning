# CHIP-8 中文学习指导

本目录用于配合仓库中的 CHIP-8 参考源码和测试套件学习，目标不是直接复制现成代码，而是用 C 语言从零理解并实现一个可运行、可测试的 CHIP-8 模拟器。

## 仓库内相关目录

| 目录 | 用途 | 使用方式 |
|---|---|---|
| `../../01_chip8_reference/` | C + SDL2 的完整参考实现 | 阅读和对照，不建议直接修改 |
| `../../01_chip8_tests/` | Timendus CHIP-8 测试 ROM 套件 | 用于分阶段验证自己的实现 |
| `../../00_build-your-own-x/` | Build Your Own X 教程索引 | 暂时只作资料查询 |

## 最终学习目标

完成后应能独立解释并实现：

1. CHIP-8 的内存、寄存器、程序计数器和栈；
2. 取指、译码、执行循环；
3. 35 条左右的基础 CHIP-8 指令；
4. 64×32 单色显示和 XOR 精灵绘制；
5. 16 键键盘输入；
6. 60 Hz 延时/声音定时器；
7. ROM 文件加载；
8. 使用测试 ROM 定位指令错误。

## 推荐学习顺序

### 阶段 0：准备开发环境

目标：能够在 Windows 上编译普通 C 程序，再安装 SDL2。

建议工具：

- Windows 11；
- 64 位 PowerShell；
- MSYS2 MinGW 64-bit；
- GCC、Make、GDB；
- SDL2；
- VS Code（可选）。

先完成一个最简单的 `hello.c` 编译，不要一开始就处理窗口、声音和完整参考源码。

### 阶段 1：理解 CHIP-8 的虚拟硬件

阅读：[`01-CHIP8基础概念.md`](./01-CHIP8基础概念.md)

重点理解：

- 4 KB 内存；
- `V0`～`VF` 16 个 8 位寄存器；
- `I` 地址寄存器；
- `PC` 程序计数器；
- `SP` 和调用栈；
- 显示缓冲区；
- 键盘状态；
- 延时与声音定时器。

对应参考源码：

- `../../01_chip8_reference/chip8.h`

验收标准：不看资料也能说出 `PC`、`I`、`VF`、`SP` 各自作用。

### 阶段 2：初始化与 ROM 加载

重点理解：

- 为什么整个虚拟机状态要清零；
- 为什么 ROM 从 `0x200` 开始加载；
- 字体数据为什么放入低地址内存；
- 为什么 `PC` 初始值是 `0x200`。

对应参考源码：

- `chip8.c` 中的字体数组；
- `Chip8_New()`；
- `main.c` 中的 `ReadFile()`。

验收标准：自己实现初始化函数和 ROM 加载函数，并打印 ROM 大小、起始地址和前几字节。

### 阶段 3：实现取指、译码、执行循环

一条 CHIP-8 指令固定为 2 字节，典型取指方式：

```c
uint16_t opcode =
    ((uint16_t)memory[pc] << 8) |
    memory[pc + 1];

pc += 2;
```

然后根据指令高 4 位译码：

```c
switch (opcode & 0xF000) {
case 0x1000:
    /* 1NNN：跳转 */
    break;
case 0x6000:
    /* 6XNN：给 VX 赋值 */
    break;
}
```

验收标准：手工写入几条指令，能够正确打印取出的操作码和字段。

### 阶段 4：先实现最小指令集

优先顺序：

1. `00E0`：清屏；
2. `1NNN`：跳转；
3. `6XNN`：给寄存器赋立即数；
4. `7XNN`：寄存器加立即数；
5. `ANNN`：设置地址寄存器；
6. `DXYN`：绘制精灵。

完成这些指令后，先运行测试套件中的：

- `1-chip8-logo.ch8`；
- `2-ibm-logo.ch8`。

### 阶段 5：补齐控制流与算术指令

实现：

- 条件跳过：`3XNN`、`4XNN`、`5XY0`、`9XY0`；
- 子程序：`2NNN`、`00EE`；
- 逻辑和算术：`8XY0`～`8XYE`；
- 随机数：`CXNN`；
- 带 `V0` 偏移跳转：`BNNN`。

验收测试：

- `3-corax+.ch8`；
- `4-flags.ch8`。

### 阶段 6：显示、键盘和定时器

实现：

- 64×32 显示缓冲区；
- 精灵 XOR 绘制；
- 碰撞时设置 `VF = 1`；
- `EX9E`、`EXA1` 键盘指令；
- `FX0A` 等待按键；
- 60 Hz 的 `delay_timer` 和 `sound_timer`。

验收测试：

- 键盘测试；
- Beep 测试；
- Quirks 测试。

### 阶段 7：SDL2 平台层

最后再阅读 `../../01_chip8_reference/main.c`，学习：

- SDL 窗口和纹理；
- 电脑键盘到 CHIP-8 键位的映射；
- 屏幕刷新；
- 声音设备；
- 主循环和执行频率。

核心原则：

> CHIP-8 核心逻辑不要依赖 SDL2。SDL2 只负责窗口、输入、显示和声音。

### 阶段 8：完整测试与整理

阅读：[`04-测试与验收.md`](./04-测试与验收.md)

最终要求：

- 基础测试 ROM 可以通过；
- 不依赖参考项目也能编译自己的项目；
- 每个指令函数都有明确注释；
- 能打印 PC、opcode、寄存器和栈，方便调试；
- README 记录构建和运行方法。

## 学习原则

- 一次只实现一小部分；
- 每完成一条指令就做单元验证；
- 先写无 SDL2 的 CPU 核心，再写窗口层；
- 参考源码用来对照，不直接整段复制；
- 遇到错误先打印 `PC`、`opcode`、`V0`～`VF`、`I` 和 `SP`；
- 测试 ROM 失败时，先定位是哪组 opcode，而不是盲目修改。
