# 02：CHIP-8 参考源码阅读路线

本篇直接对应仓库中的：

```text
01_chip8_reference/
```

不要从 `main.c` 第一行一路读到最后。推荐按“核心状态 → 初始化 → 指令循环 → 显示 → 平台层”的顺序阅读。

## 一、先读 `chip8.h`

路径：

```text
../../01_chip8_reference/chip8.h
```

### 1. 基础类型

```c
typedef uint8_t u8;
typedef uint16_t u16;
typedef uint32_t u32;
```

含义：

- `u8`：8 位无符号整数；
- `u16`：16 位无符号整数；
- `u32`：32 位无符号整数。

CHIP-8 的寄存器多为 8 位，指令和地址通常使用 16 位保存。

### 2. 屏幕尺寸

```c
#define SCREEN_WIDTH 64
#define SCREEN_HEIGHT 32
```

这两个宏是 CHIP-8 固定显示分辨率。

### 3. `chip8_vm` 结构体

重点字段：

```c
u8 ram[4096];
u8 v[16];
u16 i;
u8 delayTimer;
u8 soundTimer;
u16 pc;
u8 sp;
bool keystate[16];
```

建议阅读时自己画表：

| 字段 | 中文含义 | 位宽/数量 |
|---|---|---|
| `ram` | 内存 | 4096 字节 |
| `v` | V0～VF 寄存器 | 16×8 位 |
| `i` | 地址寄存器 | 16 位保存 |
| `pc` | 程序计数器 | 16 位保存 |
| `sp` | 栈指针 | 8 位保存 |
| `delayTimer` | 延时定时器 | 8 位 |
| `soundTimer` | 声音定时器 | 8 位 |
| `keystate` | 16 个按键状态 | 16 个布尔值 |

### 4. 参考实现的特殊设计

```c
u8 *screen;
u16 *_stack;
```

参考实现不是单独申请屏幕和栈，而是让它们指向 `ram` 的后部区域：

```text
screen → ram[0xF00]
stack  → ram[0xEA0]
```

初学者自己的版本可以先使用独立数组：

```c
uint8_t display[64 * 32];
uint16_t stack[16];
```

等功能正确后，再理解位压缩和共享内存布局。

### 5. 对外函数

```c
chip8_vm *Chip8_New(...);
void Chip8_Tick(chip8_vm *vm);
int Chip8_GetPixel(...);
```

对应：

- `Chip8_New`：创建并初始化虚拟机；
- `Chip8_Tick`：执行一个 CPU 周期；
- `Chip8_GetPixel`：读取某个屏幕像素。

读完 `chip8.h` 后先停止，确保你能解释结构体中的每个字段。

## 二、再读 `chip8.c` 的初始化部分

路径：

```text
../../01_chip8_reference/chip8.c
```

先只看文件开头到 `Chip8_New()` 结束。

### 1. `fontROM`

这是 0～F 的内置字体精灵。每个字符使用 5 个字节，每行使用一个字节的高 4 位表示像素。

例如数字 `0`：

```c
0xF0,
0x90,
0x90,
0x90,
0xF0
```

二进制形式：

```text
1111
1001
1001
1001
1111
```

### 2. `Chip8_GetPixel()` 与 `SetPixel()`

参考实现一个字节保存 8 个像素，所以需要计算：

```text
像素总序号 = y × 64 + x
字节序号   = 像素总序号 ÷ 8
位偏移     = 像素总序号 % 8
```

这是很好的位运算练习，但不是第一版必须采用的方式。

### 3. `ClearScreen()`

清屏本质就是把显示缓冲区全部置 0。

### 4. `Chip8_New()`

按以下顺序阅读：

1. 给虚拟机结构体申请内存；
2. 检查 ROM 是否过大；
3. `memset` 清零；
4. 把 ROM 复制到 `ram + 0x200`；
5. 把字体复制到低地址；
6. 设置屏幕和栈位置；
7. 设置 `pc = 0x200`；
8. 保存随机数函数；
9. 清屏。

你自己的初始化函数也应保持类似顺序。

## 三、阅读操作码函数

`chip8.c` 中的：

```text
Op_0
Op_1
Op_2
...
Op_F
```

是按照指令最高 4 位分组的。

### 推荐阅读顺序

不要按文件顺序全部读完，先读最简单的一组：

1. `Op_6`：`6XNN`，立即数赋值；
2. `Op_7`：`7XNN`，立即数加法；
3. `Op_A`：`ANNN`，设置 `I`；
4. `Op_1`：`1NNN`，跳转；
5. `Op_3`、`Op_4`：条件跳过；
6. `Op_2`、`Op_0`：调用和返回；
7. `Op_8`：寄存器算术和位运算；
8. `Op_D`：精灵绘制；
9. `Op_E`：键盘；
10. `Op_F`：定时器、字体、BCD、内存传输。

### 先读 `Op_6`

```c
vm->v[(ins >> 8) & 0xf] = ins & 0xff;
```

把它拆开：

```c
int x = (ins >> 8) & 0x0F;
uint8_t nn = ins & 0x00FF;
vm->v[x] = nn;
```

后者更适合初学阶段调试。

### 再读 `Op_8`

这一组最低 4 位决定具体运算：

```text
8XY0
8XY1
8XY2
...
8XYE
```

注意 `VF` 的设置时机。先保存原始 `VX` 和 `VY`，再写结果，通常更安全。

### 最后读 `Op_D`

这是整个 CHIP-8 最难的基础指令之一，涉及：

- 从 `I` 指向的内存读取精灵；
- 每行 8 个像素；
- XOR；
- 屏幕边界；
- 碰撞检测；
- `VF` 标志。

建议自己先写伪代码，再看参考实现。

## 四、找到并读 `Chip8_Tick()`

使用编辑器搜索：

```text
Chip8_Tick
```

阅读时重点找出四件事：

1. 如何从 `ram[pc]` 和 `ram[pc + 1]` 组成 16 位指令；
2. `pc` 在何时加 2；
3. 如何根据最高 4 位调用 `Op_0`～`Op_F`；
4. 遇到未知指令如何报错或停止。

建议把它改写成自己的伪代码：

```text
如果虚拟机停止，则返回
如果正在等待按键，则返回
读取两个字节组成 opcode
PC 前进两个字节
读取 opcode 的最高半字节
调用对应指令处理函数
```

`Chip8_Tick()` 是 CPU 核心，必须逐行理解。

## 五、最后读 `main.c`

路径：

```text
../../01_chip8_reference/main.c
```

`main.c` 主要不是 CPU，而是平台层。

### 推荐函数顺序

1. `ReadFile()`：读取 ROM；
2. `RandomNumber()`：为 `CXNN` 提供随机数；
3. `MapForChip8()`：键盘映射；
4. `ProcessEvents()`：处理按键和退出；
5. `UpdateScreen()`：把 CHIP-8 像素传给 SDL；
6. `InitSDL()`：创建窗口、渲染器、纹理和声音设备；
7. `UpdateAudio()`：根据声音定时器产生声音；
8. `main()`：组合所有模块并运行主循环。

### 需要特别注意

参考源码中平台逻辑和文件读取都集中在 `main.c`。你自己的项目可以进一步拆分：

```text
chip8.c          CPU 和虚拟硬件核心
platform_sdl.c   SDL2 窗口、键盘、声音
rom.c            ROM 文件读取
main.c           程序入口和主循环
```
